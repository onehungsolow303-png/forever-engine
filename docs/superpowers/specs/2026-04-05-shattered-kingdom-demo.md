# Shattered Kingdom — Playable Demo Design Specification

**Date:** 2026-04-05
**Status:** Approved
**Type:** Vertical Slice Demo (~30 minutes gameplay)
**Genre:** Strategy/RPG Hybrid + Sandbox Survival
**Setting:** Dark fantasy post-apocalyptic cursed kingdom

---

## 1. Vision

A lone wanderer explores a procedurally generated hex overworld in a cursed, ruined kingdom. Survival mechanics (hunger, thirst, day/night) create urgency. Discoverable locations (towns, dungeons, castles, fortresses) are entered as separate scenes. Combat is turn-based on generated tactical battle maps using D&D 5e mechanics. A 4-quest chain provides narrative structure. AI systems adapt to the player's style throughout.

Single-player demo, architected for multiplayer expansion. Windows PC build. Open world expandable beyond the demo region.

## 2. Core Game Loop

```
Hex Overworld → discover/explore/survive
    ↓ enter location tile OR random encounter triggers
Battle Map Scene → turn-based tactical RPG combat
    ↓ victory → loot | defeat → respawn at last safe point
Return to Overworld → spend loot, progress quests, keep exploring
```

## 3. Overworld Scene

### 3.1 Hex Grid
- 20x20 hex grid (400 tiles)
- Procedurally generated from seed using terrain noise
- Terrain types: Plains (1.0 move cost), Forest (2.0, +0.25 defense), Mountain (3.0, +0.5 defense), Ruins (1.5, encounter zone), Water (impassable), Road (0.5)
- Player token moves tile-to-tile, one hex per action
- Camera follows player, orthographic, zoom in/out

### 3.2 Fog of War
- Hexes start unexplored (black)
- Player reveals 2-hex radius around current position
- Explored hexes stay dimmed, visible hexes are full brightness
- Uses the existing RTSFogOfWar Burst jobs adapted for hex grid

### 3.3 Day/Night Cycle
- 10-minute real-time day cycle (5 min day, 5 min night)
- Day: normal visibility (2 hex), lower encounter rate (10%)
- Night: reduced visibility (1 hex), higher encounter rate (25%), stronger enemies
- Visual: ambient light color shifts, sun rotation

### 3.4 Survival
- Hunger: drains 1 point per 5 moves, max 100. At 0 = -1 HP per move
- Thirst: drains 1 point per 3 moves, max 100. At 0 = -2 HP per move
- Food found: at towns (buy), wilderness (forage on forest tiles, 30% chance), loot
- Water found: at towns, rivers (adjacent to water tiles), loot

### 3.5 Random Encounters
- Trigger chance per move: Plains 5%, Forest 15%, Ruins 25%, Road 0%, Night +15%
- Encounter types: bandits (plains/road), wolves (forest), undead (ruins), mutants (ruins at night)
- Encounter generates a small battle map (8x8) and transitions to Battle Scene
- AI Director modulates encounter rate based on pacing (suppress if player had 3 fights in a row, boost if 10+ moves with no action)

### 3.6 Discoverable Locations (5)

| # | Name | Type | Hex Position | Function |
|---|------|------|-------------|----------|
| 1 | Survivor's Camp | Camp | (2,2) | Start point, tutorial, rest, save |
| 2 | Ashwick Ruins | Town | (8,5) | NPC dialogue, shop, quest giver |
| 3 | The Hollow | Dungeon | (12,10) | Generated multi-room dungeon, quest target |
| 4 | Ironhold | Fortress | (5,15) | Friendly dwarves, rest, weapon upgrade |
| 5 | Throne of Rot | Castle | (17,17) | Boss encounter, final quest |

Each location transitions to a generated interior map (Battle Scene) with its own layout.

## 4. Battle Map Scene

### 4.1 Map Generation
- Random encounters: 8x8 grid, open terrain with scattered obstacles
- Dungeons: 10+ rooms, corridors, generated by PipelineCoordinator
- Castle: 12+ rooms, throne room boss chamber, generated
- Town: open layout, no combat (dialogue/shop only, same scene system)

### 4.2 Turn-Based Combat
- Initiative: all combatants roll 1d20 + DEX mod, sorted descending
- Player turn: move (up to Speed tiles), attack (1 action), or use item
- Enemy turn: AI decides via AIDecisionJob (chase/guard/flee/flank based on behavior)
- Attack: 1d20 + STR mod vs AC. Hit = roll damage dice. Miss = 0.
- Death: HP <= 0. Player death = respawn at last safe location with inventory intact.
- Victory: all enemies defeated. Loot screen shows gold + items found.

### 4.3 D&D 5e Stats
- Player starts: HP 20, AC 12, STR 14, DEX 12, CON 12, SPD 6, ATK 1d8+2
- Enemies scale with Dynamic Difficulty system
- Leveling: gain 1 level per quest completed. +5 HP, +1 to one stat per level.

### 4.4 AI Behavior
- Enemies use AIDecisionJob with behavior types from encounter definition
- Bandits: Chase (aggressive melee)
- Wolves: Pack tactics (flank if ally adjacent to target)
- Undead: Guard (hold position, attack if in range)
- Mutants: Adaptive (uses Q-learning, learns player patterns across encounters)
- Boss: Scripted phases (Phase 1: guard, Phase 2 at 50% HP: chase + enrage)

## 5. Quest Chain

### Quest 1: "Signal Fire" (Tutorial)
- **Trigger:** Game start at Survivor's Camp
- **Objective:** Reach Ashwick Ruins (navigate overworld, teaches movement + survival)
- **Reward:** 20 gold, food rations x5
- **Systems:** Overworld, survival, fog of war

### Quest 2: "The Merchant's Plea"
- **Trigger:** Talk to merchant NPC in Ashwick Ruins
- **Objective:** Clear The Hollow dungeon (defeat 5 enemies inside)
- **Reward:** 50 gold, iron sword (+2 damage), merchant stock unlocks
- **Systems:** Dialogue, dungeon generation, combat, inventory

### Quest 3: "Dwarven Alliance"
- **Trigger:** Complete Quest 2, merchant mentions the dwarves
- **Objective:** Visit Ironhold, deliver merchant's letter (item in inventory)
- **Reward:** Armor upgrade (AC +2), full rest (HP/hunger/thirst restored)
- **Systems:** Dialogue, inventory (quest item), NPC memory (dwarves remember you)

### Quest 4: "The Cursed Throne"
- **Trigger:** Complete Quest 3, dwarven chief reveals the curse source
- **Objective:** Defeat the Rot King boss in the Throne of Rot castle
- **Reward:** Victory screen, game complete
- **Systems:** Boss combat, AI Director (dramatic pacing), all systems converge

## 6. Inventory & Economy

- 20-slot inventory (existing system)
- Item types: weapons (equip to change ATK dice), armor (equip to change AC), food (eat to reduce hunger), water (drink to reduce thirst), potions (heal HP), quest items
- Gold: earned from combat loot and quests, spent at Ashwick shop
- Shop: buy food, water, potions. Sell loot items.

## 7. Player Character

- Name: "Wanderer" (hardcoded for demo)
- Token: blue circle (procedural sprite from ProceduralSpriteGenerator)
- Starts at Survivor's Camp with: rusty sword (1d6), leather armor (AC 12), food x3, water x3, 10 gold

## 8. AI Systems Integration

| System | Demo Integration |
|--------|-----------------|
| AI Director | Modulates random encounter rate, triggers dramatic events near quest locations |
| Dynamic Difficulty | Scales enemy HP/damage based on player kill/death ratio |
| Player Profiler | Tracks melee vs cautious play, adjusts encounter compositions |
| NPC Memory | Town merchant remembers if you completed quests, dialogue changes |
| Episodic Memory | "You cleared The Hollow" remembered, referenced in later dialogue |
| Self-Healing | Missing sprites get placeholder, missing audio plays silence |
| Performance Reg | Auto-reduces quality if framerate drops (large battle maps) |

## 9. Audio

- Overworld: ambient wind + birds (day), crickets + howling (night)
- Combat: hit/miss SFX, death SFX, turn start chime
- Music: exploration track (overworld), combat track (battle), town track (safe locations)
- All via AudioManager with AudioConfig ScriptableObject

## 10. Save/Load

- Quicksave: automatic on entering any safe location (camp, town, fortress)
- Manual save: pause menu → save
- Save data: player position (hex), stats, inventory, quest progress, survival meters, day count, explored hexes
- Load: main menu → continue

## 11. Scenes

| Scene | Purpose |
|-------|---------|
| `MainMenu` | Title screen, new game, continue, quit |
| `Overworld` | Hex grid exploration, survival, discovery |
| `BattleMap` | Turn-based combat (random encounters + location interiors) |
| `Town` | NPC dialogue, shop (uses BattleMap scene with no enemies) |

## 12. Success Criteria

The demo is complete when a player can:
1. Start a new game at Survivor's Camp
2. Navigate the hex overworld with fog of war revealing terrain
3. Manage hunger and thirst while exploring
4. Experience random encounters with turn-based combat
5. Visit all 5 locations and interact with NPCs
6. Complete all 4 quests in sequence
7. Defeat the final boss
8. See a victory screen
9. Save and load their progress
